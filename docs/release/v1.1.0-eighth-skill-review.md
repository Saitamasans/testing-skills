## 优点

1. 顶部硬规则直接约束最大风险：只执行已有测试用例（Test Cases），不生成用例、不猜环境、不静默转换字段、不泄露密钥。
2. `description` 同时覆盖自动执行、固定 manifest、证据生成、Excel/HTML 回填和产物验证等自然触发意图，并明确排除用例生成、需求澄清和用例审计，互斥路由清楚。
3. 输入、风险、定位器与断言、Runner 命令、CI 报告拆成五份按需引用，主文件保持在 122 行，没有把所有分支一次性塞进上下文。
4. 执行前强制展示主/辅 Skill、manifest hash、目标 origin、风险、动作数量和环境变量名；用户确认前禁止 Web/API/数据库动作。
5. 交互执行完整展示执行准备、用例预告、实时执行、证据收集和结果中心；正式 Web PNG 隐藏驾驶舱，桌面录屏保留完整视觉过程。
6. 最终自检和反模式明确保留“未执行、通过、不通过、待定”四状态，避免把待定误算成研发 Bug。
