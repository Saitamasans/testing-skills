## Skill-review audit

Target: `skills/web-api-test-execution-evidence`.

Strengths:

- Structure is standard: root `SKILL.md`, `agents/openai.yaml`, one-level `references/`, executable `scripts/` and the locked release manifest under `assets/`.
- Generated `SKILL.md` is 111 lines, below the 500-line limit.
- Description starts with `Use when` and includes execution, evidence, backfill, local/CI and negative trigger boundaries.

Accepted findings:

- Install scripts and validation output still referenced the old seven-Skill scope. Fixed by adding `web-api-test-execution-evidence` to both install-all scripts and updating validation text.
- Generated `SKILL.md` appended generic reference-loading footer lines after each embedded reference. Fixed by keeping the more specific section-level loading conditions and adding a regression test.
- The eighth Skill source file was visually mixed with the original seven root-level sources. Fixed by moving only the eighth source under `skill-sources/web-api-test-execution-evidence/`; the original seven root source filenames remain unchanged.
- Runner approval examples used a far-future expiry. Fixed by using `<ISO_EXPIRES_AT>` and explicitly requiring a short-lived execution window.

Rejected findings:

- No additional reference extraction needed. The generated `SKILL.md` is already short, and detailed protocols are in five focused references.
