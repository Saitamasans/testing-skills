# 真实桌面录制版八 Skill 演示设计

## 目标

重录一版可对外发布的真实操作视频：观众能看到 Codex 中输入七个 Skill 的调用口令、第八个 Skill 的执行预览与确认、真实 Web/API 操作和最终报告回填。现有静态材料导航视频保留为辅助索引，不再作为主演示视频。

## 录制边界

- 只使用本地隔离 SkillMart 演示系统和当前测试材料，不访问真实生产系统、账号、Token 或个人窗口。
- 录制 Codex 主窗口、内置终端和内置浏览器的真实画面，包含鼠标、键盘输入、等待返回、点击和页面变化。
- 七个 Skill 分七次独立调用，每章必须至少出现一次真实输入和一次真实输出。
- 第八个 Skill 必须完整展示：输入、准备度检查、执行预览、审批确认、依赖准备、Web/API 执行、断言、状态回填和报告生成。
- 五套执行真实运行；视频聚焦代表性过程，同时显示五套最终汇总和独立证据。
- 原始录屏为连续桌面录制；精剪版保留真实操作顺序，压缩等待和重复浏览，时长不超过 20 分钟。

## 录制章节

1. 环境与演示资料：展示本地 PRD、接口契约、执行清单和隔离地址。
2. 需求澄清：现场输入 `requirement-clarification-test`，等待 P0 阻塞结论。
3. 需求工作台：现场输入 `requirement-test-workbench`，打开正式十列测试用例（Test Cases）。
4. 单接口完整版：现场输入 `single-api-test-full`，展示 POST `/api/orders` 分析。
5. 单接口精炼版：现场输入精炼版口令，展示短上下文输出。
6. 多接口链路：现场输入 `multi-api-flow-test`，展示字段依赖和链路断言。
7. 正式服验证：现场声明线上验证目标，展示缺少生产写入门禁时的 L0 只读拦截。
8. 测试用例审计：现场输入 `test-case-quality-audit`，展示问题、待定和修订映射。
9. 第八个 Skill：真实执行一套 Web 成功、一套 Web 失败、一套 API 链路，并完成五套汇总。
10. 交付验收：打开独立 PNG、HTML、Excel、run-result、日志、Trace、原始录屏和精剪版。

## 通过标准

- 视频画面中不得以“下一页”播放器代替实际 Codex/终端/浏览器操作。
- 第八个 Skill 章节中必须可见审批确认和至少一个真实页面状态变化，例如订单号从无到有、第二次点击生成重复订单或库存断言失败。
- 录制脚本、操作日志和章节时间轴必须能对应到实际画面，不得只根据预生成文件拼接叙事。
- 重新执行材料门禁、Runner 回归、视频解码和隐私扫描；任何一项失败都不发布主视频。
