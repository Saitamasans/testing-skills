# SkillMart PRD v1

## 业务范围

SkillMart 提供商品列表、优惠券校验、创建订单、支付回调、订单查询和取消订单能力。演示系统只使用本地内存数据，不连接真实生产系统、真实支付、真实用户和真实数据库。

## 已确认规则

- 普通用户只能处理自己的订单。
- 商品库存大于 0 才能创建订单，订单创建时锁定库存。
- `SKILL20` 优惠券满 100 减 20。
- 创建订单必须携带幂等键；重复请求只能生成一张订单。
- 支付回调必须幂等。
- 待支付订单可以取消并释放库存。
- 已支付、已取消订单不可取消。
- 页面、接口、订单状态和库存结果必须一致。

## 故意保留的真实缺陷

`POST /api/orders` 暂未正确处理幂等键。相同用户使用相同幂等键提交同一订单时，系统会创建两张订单并重复锁定库存。对应测试用例（Test Case）应真实执行并判定为 `不通过`。

## 故意保留的需求歧义

优惠券刚好跨过过期边界时，产品确认记录与接口契约存在时间口径冲突。对应测试用例（Test Case）应真实执行并判定为 `待定`。
