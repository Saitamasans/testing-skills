## 十五、表格文件完整版规范

表格文件用于编辑、评审、交付、归档。
文件生成条件统一见“二十四、最终输出要求”。

### 15.1 文件命名规则

默认命名：

`接口名称_单接口测试用例_YYYYMMDD.xlsx`

如有模块名：

`模块名_接口名称_单接口测试用例_YYYYMMDD.xlsx`

如多次生成，追加版本号：

`接口名称_单接口测试用例_YYYYMMDD_v1.xlsx`

| Sheet 名称  | 是否必须 | 用途                                    |

## 十八、测试用例编写规则

### 18.1 用例标题规则

用例标题格式建议：

`【类型】验证目标`

示例：

* 【正向】合法用户获取钻石余额成功
* 【鉴权】未传 Authorization 时拒绝访问
* 【参数】缺失必填参数时返回失败
* 【资产】余额为 0 时返回 diamond=0
* 【越权】用户不能查询其他用户资产
* 【并发】同一用户并发查询余额结果稳定

### 18.2 前置条件规则

前置条件必须包含：

1. 测试环境。
2. 测试账号。
3. 登录态 / token。
4. 角色权限。
5. 数据准备。
6. 第三方依赖。
7. 特殊状态。
8. 无法准备的数据需备注说明。

### 18.3 测试步骤规则

测试步骤必须包含：

1. 请求方式。
2. 接口地址。
3. Header。
4. Query / Path / Body / FormData。
5. 关键测试数据。
6. 调用方式。
7. 必要的查询或校验动作。

### 18.4 预期结果规则

预期结果必须包含：

1. HTTP 状态。
2. 业务 code。
3. msg。
4. response 字段。
5. 字段类型。
6. 字段取值。
7. 数据副作用。
8. 错误响应。
9. 安全要求。
10. 若具体值缺失，必须标注待确认。

禁止出现以下模糊表达：

* 返回正常
* 系统正常
* 处理成功
* 符合预期
* 无异常
* 正确返回

必须改为：

* HTTP 状态码为 200
* 业务 code 为接口约定成功码
* 返回字段包含 `data.fieldName`
* 字段类型为 number/string/boolean
* 不返回其他用户数据
* 不产生重复扣减 / 重复发放
* 具体 code/msg 待确认

## 十九、错误码缺失时的固定表达

当错误码文档缺失时，异常用例预期统一使用以下表达：

> 接口应返回失败响应，不应返回成功业务结果；不应产生数据副作用；具体 HTTP 状态码、业务 code、msg 待产品 / 后端确认。

当鉴权错误码缺失时：

> 接口不应返回用户数据或敏感信息；应返回鉴权失败响应；具体 HTTP 状态码、业务 code、msg 待产品 / 后端确认。

当参数错误码缺失时：

> 接口应识别参数错误并返回失败响应；不应出现 500 未处理异常；具体 code/msg 待产品 / 后端确认。

当幂等规则缺失时：

> 重复请求不应造成重复扣减、重复发放、重复创建或重复状态变更；具体幂等返回规则待产品 / 后端确认。

## 二十一、资料不足时的处理边界

### 21.1 可以做的事

1. 输出资料质量判断。
2. 输出缺失信息。
3. 输出待确认问题。
4. 输出接口风险。
5. 输出轻量测试点。
6. 仅在对应场景不受阻塞型缺口影响时，输出可确认部分的用例。
7. L0 / L1 的候选内容按 6.5 输出为五列轻量测试方向。
8. 标注合理假设。
9. 标注置信度。

### 21.2 不可以做的事

1. 不得生成完整正式用例并声称可直接执行。
2. 不得写死错误码。
3. 不得写死业务提示。
4. 不得写死枚举范围。
5. 不得写死权限边界。
6. 不得写死金额口径。
7. 不得写死状态流转。
8. 不得写死第三方回调规则。

## 二十二、风险声明模板

当资料不完整时，必须输出风险声明。

推荐表达：

> 本次用例基于用户提供的接口资料生成。文档中未明确的错误码、鉴权规则、字段枚举、业务规则、数据副作用、幂等策略或并发策略，已在待确认问题和备注中标记。未确认前，相关用例不应作为最终验收口径，只能作为测试设计候选或风险提醒。
