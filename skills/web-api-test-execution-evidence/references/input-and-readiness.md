# 输入与准备度

## 输入类型

- 正式输入：原生 `report.json`、标准十列 `.xlsx`、经确认映射的非标准 `.xlsx`。
- 非正式输入：HTML、CSV、Markdown、截图、PRD、接口文档、口头描述。非正式输入只能用于提示用户补齐材料，不能直接执行。

## 新用户资料分级

### 强制资料

- 正式测试用例：标准十列 `.xlsx`、原生 `report.json`，或已确认字段映射的非标准 Excel；用例必须包含可执行步骤和预期结果。
- 目标 Web/API 地址：必须能从当前执行环境访问。
- 环境性质和执行授权：明确测试、预发或正式环境，不根据域名猜测；执行方必须有对应授权。
- 执行前确认：先展示执行预览、目标 origin、风险等级、动作数量和凭据变量名，再由用户确认。
- Node.js 20+：实际执行 Runner 时必须可用；不要求 npm、npx 或 npm 账号。

### 条件强制资料

- 需要登录时：提供测试账号和凭据来源；秘密值使用环境变量或 Secrets，不写入用例、配置或报告。
- 接口用例缺少可执行调用细节时：提供接口文档、OpenAPI、Swagger 或 Postman，补齐方法、路径、参数、请求体、鉴权和断言。
- 会新增或修改数据时：提供隔离的测试数据和清理方案；涉及删除时同时按风险规则审批。
- Web 用例无法用稳定语义定位时：提供可靠的角色、标签、test id 或其他稳定控件信息。
- 包含数据库断言时：提供只读账号、只读证明和对应驱动，只允许 `SELECT`。
- 本地应用尚未运行时：提供所需前后端源码、启动命令、依赖和配置；已有可访问测试地址时通常不需要前后端源码。
- CI 执行时：提供 CI 配置和 Secrets，且 CI 只允许 R0/R1。

### 辅助资料

- 需求文档、需求截图、原型和流程图：用于核对业务口径，不能代替正式测试用例。
- 前后端源码：已有可访问地址时仅在定位控件或排查失败原因时按需提供。
- 历史报告和旧证据：用于回归对比，不是首次执行门槛。

需求文档、需求截图、原型和流程图不能代替正式测试用例。

只有需求资料而没有正式测试用例时，不得直接执行。先建议调用 `requirement-test-workbench` 生成 Excel 正式用例；待用例、目标和授权齐备后，再回到本 Skill。

## 准备度 E0-E4

- E0：没有可执行用例或执行目标。
- E1：有用例，但缺目标、账号、数据、审批或字段映射。
- E2：材料基本齐，但存在执行风险或环境/数据疑点。
- E3：可以生成执行预览，等待用户确认。
- E4：manifest、approval、凭据来源和输出目录已锁定，可以执行。

## 字段映射

非标准 Excel 每次都展示字段映射预览，必须由用户确认。AI 置信度再高也不能静默转换。映射确认后再生成 manifest。

## 提示模板

先说“可以继续/暂时不能继续”，再列已具备、缺失材料、可选材料。缺失项给具体例子，如 `TESTING_API_TOKEN`、`execution-profile.json`、只读数据库账号、清理接口。
